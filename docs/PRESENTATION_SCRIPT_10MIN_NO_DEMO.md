## 슬라이드 1. 표지 & 한 줄 메시지 (0:35)

### 슬라이드 구성
- 타이틀: `Synapse`
- 서브타이틀: `시청기록으로 나를 진단하고 이상향을 설계하다`
- 중앙 강조 문장(한 줄): 핵심 메시지
- 하단: 팀명/발표자

### 발표 스크립트
안녕하세요. 저희가 만든 Synapse는 한 문장으로 설명할 수 있습니다.  
"당신의 유튜브 시청기록이 곧 당신이다."  
Synapse는 이 기록을 단순 소비 로그로 보지 않고, **지금의 나를 진단하고, 되고 싶은 나를 설계해, 실제 행동으로 연결하는 플랫폼**입니다.  
오늘은 이 흐름이 어떻게 제품으로 구현됐는지 10분 안에 명확하게 보여드리겠습니다.

### 근거
- 프로젝트 한 줄 정의: [`docs/OVERVIEW.md`](./OVERVIEW.md)

## 슬라이드 2. 문제 정의: 왜 필요한가 (0:55)

### 슬라이드 구성
- 좌측: 문제 키워드 3개  
  - `수동 소비`  
  - `자기이해 부재`  
  - `의도적 성장의 단절`
- 우측: 질문 한 줄  
  - `우리는 많이 보지만, 그 데이터로 나를 이해하고 있는가?`

### 발표 스크립트
많은 사용자가 유튜브와 웹에서 많은 시간을 보냅니다.  
문제는 소비량이 아니라, **그 기록이 자기이해로 연결되지 않는다**는 점입니다.  
알고리즘이 추천한 것을 계속 소비하지만, 내 관심·가치·행동 패턴을 구조적으로 읽어내지는 못합니다.  
결국 "많이 봤다"는 사실은 남는데, "어떤 방향으로 변하고 있는가"는 남지 않습니다.  
저희는 이 단절을 해결하기 위해, 소비 기록을 **진단 가능한 데이터**로 바꾸는 시스템을 설계했습니다.

### 근거
- 제품 철학/문제의식: [`docs/OVERVIEW.md`](./OVERVIEW.md) `제품 철학`, `핵심 차별점`

## 슬라이드 3. Synapse 정의 (제품 한 장 정리) (0:50)

### 슬라이드 구성
- 중앙 파이프라인 한 줄:
  - `수집 → 21축 진단 → 이상향 설계 → 실행(재생목록/가이드)`
- 하단 보조 문장:
  - `웹앱 + 브라우저 확장 + 백엔드 멀티에이전트`

### 발표 스크립트
Synapse는 네 단계로 동작합니다.  
첫째, 시청 기록과 웹 활동 데이터를 수집합니다.  
둘째, 21축 성향 모델로 현재 상태를 진단합니다.  
셋째, AI와 대화하며 이상향을 설계합니다.  
넷째, 행동 가이드와 재생목록으로 실제 실행까지 연결합니다.  
즉, 분석에서 끝나는 도구가 아니라 **진단-설계-실행의 닫힌 루프**를 제공합니다.

### 근거
- 핵심 사용자 여정/한 장 요약: [`docs/OVERVIEW.md`](./OVERVIEW.md)

## 슬라이드 4. 사용자 여정 End-to-End (1:00)

### 슬라이드 구성
- `개인 분석 루프`와 `확장/거시 분석 루프`를 2단 다이어그램으로 표시
- 키 노드: `Takeout/Drive`, `Indexer`, `Profiler`, `Navigator`, `Curator`, `Archiver`, `Aggregator`, `Reporter`

### 발표 스크립트
사용자 여정을 엔드투엔드로 보면, 시작점은 Takeout 업로드 또는 Drive 자동분석입니다.  
Indexer가 기록을 정제하고, Profiler가 21축 성향 스냅샷을 만듭니다.  
그 결과를 바탕으로 Navigator는 이상향과 재생목록을 설계하고, Curator는 홈에서 대화형 탐색을 제공합니다.  
동시에 확장에서는 Archiver가 페이지 맥락 Q&A를 수행하고, 행동 로그와 스크랩 데이터는 Aggregator로 들어가 거시 트렌드로 집계됩니다.  
마지막으로 Reporter가 이를 지식 그래프와 리포트로 변환합니다.  
즉, 개인 루프와 거시 루프가 같은 데이터 철학 위에서 함께 돌아가는 구조입니다.

### 근거
- 사용자 여정 다이어그램: [`docs/OVERVIEW.md`](./OVERVIEW.md) `핵심 사용자 여정`

## 슬라이드 5. 21축 성향 모델 (핵심 차별점) (1:10)

### 슬라이드 구성
- 표 3행:
  - Schwartz 가치관 10
  - TCI 기질 3
  - 행동 스파이더 8
- 우측 작은 박스:
  - `산출물: video_analysis / user_profile_history`
  - `페르소나 라벨 + Portrait`

### 발표 스크립트
Synapse의 핵심 차별점은 21축 모델입니다.  
구성은 세 층입니다. 첫째, Schwartz 가치관 10축. 둘째, TCI 기질 3축. 셋째, 행동 스파이더 8축입니다.  
`OVERVIEW.md` 기준으로 행동 8축은 가치·기질을 기반으로 규칙과 보정 로직을 통해 도출됩니다.  
이 결과는 단순 점수표가 아니라, 영상별 의미 분석 결과 `video_analysis`와 사용자 스냅샷 `user_profile_history`로 저장됩니다.  
또한 상위 축 조합으로 페르소나 라벨과 서술형 Portrait를 만들어, 사용자가 자신의 변화를 이해하기 쉽게 제공합니다.  
그래서 Synapse는 추천 시스템이 아니라, **자기 인식 모델**에 가깝습니다.

### 근거
- 21축 정의/키/산출물: [`docs/OVERVIEW.md`](./OVERVIEW.md) `21축 성향 모델`

## 슬라이드 6. 멀티에이전트 아키텍처 개요 (1:00)

### 슬라이드 구성
- 좌: 입력 소스 (Takeout/Drive/Extension)
- 중: 7개 에이전트 블록
- 우: 출력 (개인 결과/재생목록/B2B 리포트)
- 하단 태그: `LangGraph 상태머신`, `SSE 스트리밍`

### 발표 스크립트
아키텍처는 3앱 모노레포 위에서 동작합니다.  
백엔드는 FastAPI 레이어 구조 위에 7개 에이전트가 얹혀 있고, 프론트와 확장이 같은 API에 연결됩니다.  
핵심은 상태가 있는 워크플로우를 다룬다는 점입니다.  
`OVERVIEW.md`에도 명시된 것처럼, 에이전트 파이프라인은 단계/분기/재시도를 포함하고, 사용자 대화는 SSE 스트리밍으로 즉시 반응합니다.  
즉, 단일 챗봇이 아니라 **역할이 분리된 멀티에이전트 오케스트레이션**입니다.

### 근거
- 모노레포/레이어/7에이전트/스트리밍: [`docs/OVERVIEW.md`](./OVERVIEW.md) `전체 아키텍처`, `핵심 기능 1`

## 슬라이드 7. 에이전트 상세 1: Indexer / Profiler / Navigator (1:20)

### 슬라이드 구성
- 3열 카드:
  - Indexer: `preprocess → diff → enrich → embed`
  - Profiler: `video_summary → build_profile → notify`
  - Navigator: `propose/chat/apply/guide/playlist`
- 하단: 관련 UI 경로(`/upload`, `/me/analyses`, `/me/ideals`, `/me/playlists`)

### 발표 스크립트
개인 사용자 가치의 중심은 Indexer, Profiler, Navigator입니다.  
Indexer는 Takeout을 파싱해 시청 정본을 만들고, 증분 diff와 임베딩까지 처리합니다.  
Profiler는 영상 의미 분석을 거쳐 21축 스냅샷을 생성하고, 사용자가 비교 가능한 형태로 저장합니다.  
Navigator는 그 결과를 바탕으로 이상향 3안을 제안하고, 대화형 설계와 gap 비교, 행동 가이드, 재생목록 생성까지 수행합니다.  
정리하면 데이터 파이프라인이 "수집-분석-설계-실행"으로 일관되게 이어지도록 세 에이전트가 분업합니다.

### 근거
- 각 에이전트 역할/파이프라인/API/UI: [`docs/OVERVIEW.md`](./OVERVIEW.md) `핵심 기능 1: 7개 AI 에이전트`

## 슬라이드 8. 에이전트 상세 2: Curator / Archiver (0:55)

### 슬라이드 구성
- 좌: Curator (홈 채팅)
  - 툴 오케스트레이션 6개
- 우: Archiver (확장 사이드패널)
  - `router → collect/rag/search → evaluator`
- 하단: 공통점 `대화형 인터페이스 + 데이터 근거 기반 응답`

### 발표 스크립트
Curator와 Archiver는 사용자 접점의 대화 에이전트입니다.  
Curator는 홈 채팅에서 DB 조회, 검색, 재생목록 생성, 스크랩 저장 등 툴을 호출하는 실행형 오케스트레이터입니다.  
Archiver는 확장 사이드패널에서 현재 페이지 맥락과 과거 지식을 결합해 답변하며, 근거가 부족하면 재검색 루프를 수행합니다.  
두 에이전트 덕분에 Synapse는 단순 분석 대시보드가 아니라, 사용자의 일상 맥락에서 작동하는 인터랙티브 시스템이 됩니다.

### 근거
- Curator 6툴/Archiver 워크플로우: [`docs/OVERVIEW.md`](./OVERVIEW.md) `Curator`, `Archiver`

## 슬라이드 9. 거시 트렌드: Aggregator / Reporter (0:50)

### 슬라이드 구성
- 좌: Aggregator 입력 3종 (스크랩/시청/행동)
- 중: 6대 도메인 매핑
- 우: Reporter 출력 (지식 그래프/리포트)

### 발표 스크립트
Synapse는 개인 분석에만 머물지 않고, 플랫폼 차원의 거시 트렌드도 다룹니다.  
Aggregator는 스크랩, 시청, 행동 데이터를 6대 도메인으로 매핑해 일별 스냅샷을 만듭니다.  
Reporter는 이 스냅샷을 지식 그래프와 리포트로 변환합니다.  
이 구조는 `OVERVIEW.md`에서 정의된 B2C와 B2B의 이중 가치와 정확히 연결됩니다.  
즉, 같은 기반 데이터를 개인 성장과 거시 인사이트 양쪽으로 확장하는 구조입니다.

### 근거
- B2C/B2B 이중 가치, Aggregator/Reporter 역할: [`docs/OVERVIEW.md`](./OVERVIEW.md) `이중 가치`, `Aggregator`, `Reporter`

## 슬라이드 10. 웹앱/익스텐션 표면 기능 (0:45)

### 슬라이드 구성
- 좌(웹): `/upload`, `/me/analyses`, `/me/ideals`, `/me/playlists`, `/me/scraps`, `/me/activity`
- 우(확장): FAB, 사이드패널 2탭, 트래킹 토글, 인증 브릿지

### 발표 스크립트
사용자 입장에서 보면 기능 표면은 명확합니다.  
웹에서는 업로드, 분석 확인, 이상향 설계, 재생목록 실행, 스크랩/활동 조회가 하나의 허브로 연결됩니다.  
확장에서는 브라우징 중 바로 스크랩하고, 페이지 맥락 Q&A를 수행하며, opt-in 트래킹으로 활동 데이터를 축적합니다.  
또한 웹 로그인 후 AUTH_CODE 기반 연동으로 확장 세션을 연결해 같은 계정 경험을 유지합니다.

### 근거
- 웹/확장 기능 및 인증 브릿지: [`docs/OVERVIEW.md`](./OVERVIEW.md) `핵심 기능 2`, `핵심 기능 3`, `인증`

## 슬라이드 11. 데이터/인증/운영 신뢰성 (0:30)

### 슬라이드 구성
- 3개 배지:
  - `PostgreSQL + pgvector`
  - `웹 HttpOnly 쿠키 / 확장 Bearer JWT`
  - `스케줄러 기반 자동화`

### 발표 스크립트
신뢰성 측면에서 세 가지만 짚겠습니다.  
데이터는 PostgreSQL과 pgvector를 기반으로 관계형 정보와 의미 검색을 함께 처리합니다.  
인증은 웹과 확장 채널을 분리해 운영하고, AUTH_CODE 브릿지로 안전하게 연결합니다.  
또한 자동 스케줄러로 Drive 분석, 트렌드 배치, 재생목록 갱신을 지속 수행해 일회성 분석이 아니라 운영 가능한 서비스 구조를 갖췄습니다.

### 근거
- 데이터/인증/스케줄러: [`docs/OVERVIEW.md`](./OVERVIEW.md) `데이터·AI`, `인증`, `백그라운드 스케줄러`

## 4) Q&A 대비 핵심 답변 (근거형)

### Q1. 왜 "시청기록이 곧 나"라고 말할 수 있나?
- 답변: Synapse는 시청기록을 `video_analysis`와 `user_profile_history`로 구조화해 21축 성향으로 해석한다. 즉 단순 조회가 아니라 모델 기반 진단이다.

### Q2. 왜 에이전트를 나눴나?
- 답변: `OVERVIEW.md` 기준으로 수집(Indexer), 분석(Profiler), 설계/실행(Navigator), 대화형 실행(Curator), 맥락 Q&A(Archiver), 거시 집계(Aggregator), 리포트(Reporter) 역할이 분리되어 있어 책임 경계가 명확하다.

### Q3. 개인 서비스인데 B2B를 왜 같이 다루나?
- 답변: 문서에 명시된 이중 가치 구조다. 같은 데이터 파이프라인을 기반으로 개인 성장 루프와 비식별 거시 트렌드 리포팅을 동시에 만든다.

### Q4. 확장은 왜 필요한가?
- 답변: 웹앱만으로는 업로드 시점 데이터 중심이지만, 확장은 브라우징 순간의 맥락 Q&A/스크랩/트래킹을 수집해 데이터 연속성을 만든다.
