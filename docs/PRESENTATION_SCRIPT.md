# Synapse 팀 발표 스크립트

> **형식**: 팀 발표 3인 · **덱 분량**: 약 7분 (슬라이드 1~8, 10) + 시연영상 3분 (슬라이드 9) = **총 10분**  
> **핵심 메시지**: *"당신의 유튜브 시청기록이 곧 당신이다 — Synapse는 그 데이터로 지금의 나를 진단하고, 되고 싶은 나(이상향)를 설계해 실제 행동으로 연결한다."*

---

## 발표자 배정 (이름은 팀에 맞게 교체)

| 역할 | 담당 슬라이드 | 8번 세부 갈래 |
|------|---------------|---------------|
| **발표자 A** (대표) | 1 ~ 7 | — |
| **발표자 B** | 8-0 오버뷰 + **8-1 이상향 파이프라인** | 개인 진단·이상향 (본체) |
| **발표자 C** | **8-2 트렌드 Aggregator** | 집단 트렌드 |
| **발표자 D** → *3인팀이면 C가 8-2, B가 8-1, A가 8-0 후 넘김* | **8-3 Archiver 확장** | 일상 수집 |
| **전원** | 10, 11 | — |

> 3인팀 기준: **A = 대표(1~7, 8-0)**, **B = 8-1**, **C = 8-2 + 8-3** 또는 **B = 8-1, C = 8-2, A가 8-3 짧게** — 아래 스크립트는 **B=8-1, C=8-2, A=8-3** 분담도 표기합니다. 팀 역할에 맞게 이름만 바꿔 쓰세요.

## 덱 디자인 시스템 (먼저 통일)

슬라이드 완성도를 올리려면, 내용보다 먼저 **시각 규칙**을 고정하는 게 좋습니다.

### 컬러·톤

- **배경**: 딥 네이비/차콜 계열 단색 (`#0B1020` 또는 `#111827`)
- **메인 포인트**: Synapse 블루 (`#4F8CFF`)
- **서브 포인트**: 민트 (`#35D0BA`) 1개만 추가
- **경고/문제 강조**: 레드 계열은 최소 사용 (`#FF6B6B`) — 3번 문제 슬라이드에만

### 타이포 규칙

- 제목: 44~52pt, Bold
- 본문: 24~28pt, Medium
- 캡션/각주: 16~18pt
- 한 슬라이드 텍스트량: **최대 35~40단어** 권장 (설명은 말로)

### 레이아웃 규칙

- 16:9 기준 좌우 여백 넉넉히 (좌우 6~8%)
- 기본 그리드: `2열(텍스트+비주얼)` 또는 `1열(메시지 중심)`만 사용
- 카드/박스는 그림자보다 **얇은 외곽선** 사용 (기술 발표 톤 유지)

### 애니메이션 규칙

- 전환 효과는 `Fade` 통일
- 한 슬라이드 최대 애니메이션 2개
- 요소 등장 순서: **문제/맥락 → 핵심 문장 → 결론**

### 아이콘/이미지 규칙

- 아이콘 스타일은 한 세트로 통일 (outline 또는 filled 중 택1)
- 스크린샷은 테두리+캡션 부여 (어느 화면인지 즉시 이해)
- 로고/브랜드 요소는 1~2개만 반복 노출

## 슬라이드 1 · 표지 (0:25) — 발표자 A

### 슬라이드에 보여줄 것

- 타이틀: **Synapse — 시청기록으로 나를 진단하고 이상향을 설계하다**
- 부제: 팀명 · 발표자 3인 이름
- 임팩트 비주얼 1장 (뇌 시냅스 + 데이터 흐름 느낌, 또는 제품 스크린샷 몽타주)

### 디자인 구상 (디테일)

- 레이아웃: **좌측 텍스트 40% / 우측 비주얼 60%**
- 타이틀은 2줄로 강제 개행해 리듬 만들기
- 우측 비주얼 위에 얇은 블루 그라데이션 오버레이(텍스트 가독성 확보)
- 하단에 팀명·이름은 작은 캡션으로 정렬

### 스크립트

> 안녕하세요. **[팀명]** 입니다.  
> 오늘 소개할 **Synapse**는 한 문장으로 말하면,  
> **"당신의 유튜브 시청기록이 곧 당신이다"** — 그 데이터로 **지금의 나**를 진단하고, **되고 싶은 나**를 설계해 **실제 행동**으로 연결하는 플랫폼입니다.

### 디렉션

- 첫 문장은 **천천히**, 핵심 메시지 문장은 **한 박 쉬고** 강조
- 시선은 청중 → 슬라이드 타이틀 → 다시 청중

1. 문제 정의

- 텍스트는 키워드만 남기고 부연 설명 삭제

### 스크립트

> 발표는 **문제에서 시작**해, **어떻게 만들었는지** 설명하고, **세 가지 핵심 기능**을 나눠 말씀드린 뒤, **시연 영상**으로 마무리하겠습니다.  
> 기술로 설득하고, 영상으로 증명하는 순서입니다.

### 디렉션

- 포인터로 목차만 훑고 **바로 넘어가기** — 여기서 시간 쓰지 않기

## 슬라이드 3 · 문제 정의 (0:45) — 발표자 A

### 슬라이드에 보여줄 것

- 텍스트 **최소** — 강조 문장 1개만 크게
- 보조 키워드 3개: `수동 소비` · `자기이해 부재` · `알고리즘에 끌려감`
- (선택) 하루 유튜브 시청 시간 인포그래픽

### 강조 문장 (슬라이드 중앙)

> **"우리는 하루 몇 시간을 유튜브에 쓰지만, 그게 나를 어디로 데려가는지는 모른다."**

### 디자인 구상 (디테일)

- 레이아웃: 중앙 큰 문장 + 하단 키워드 3개 배지
- 키워드 배지는 `수동 소비 / 자기이해 부재 / 알고리즘 종속` 순서
- 배경 비주얼은 흐릿한 유튜브 썸네일 모자이크(텍스트 집중 위해 blur)
- 숫자 인포그래픽 넣을 경우 한 개만: `평균 시청 시간` 또는 `누적 시청량`

### 스크립트

> 많은 사람이 하루에 **몇 시간씩** 유튜브를 봅니다.  
> 재미있고, 편하고, 알고리즘이 다음 영상을 잘 골라줍니다.  
>
> 그런데 한번 생각해 보세요. **그 시청 기록 전체를 펼쳐 놓으면** —  
> 내가 **무엇에 관심이 있고**, **어떤 가치를 우선하는지**, **어디로 성장하고 있는지** —  
> 사실 **엄청난 자기 데이터**인데, 우리는 그걸 **거의 활용하지 못합니다.**  
>
> 알고리즘이 주는 대로 보는 **수동 소비**는,  
> **자기이해**도, **의도적인 성장**도, 자연스럽게 **비어 있게** 됩니다.  
>
> 슬라이드 한 줄로 정리하면 이렇습니다.  
> **"우리는 하루 몇 시간을 유튜브에 쓰지만, 그게 나를 어디로 데려가는지는 모른다."**

### 디렉션

- 마지막 강조 문장은 **슬라이드와 동시에** 읽기
- 톤: 공감 → 문제 제기. 손가락으로 슬라이드 문장 가리키기
- 마무리 연결 멘트: "그래서 저희는 이 기록을 **문제 로그가 아닌 진단 데이터**로 다시 보기 시작했습니다."

## 슬라이드 4 · 그래서, 이런 시스템을 만들었다 (0:30) — 발표자 A

### 슬라이드에 보여줄 것

- 브릿지 문장 1줄 + **Synapse** 로고 (텍스트 최소)
- 좌우 대비 그래픽은 `소비 로그` → `자기 진단 데이터` 2개 단어만 배치

### 디자인 구상 (디테일)

- 레이아웃: 좌측 `소비 로그`(회색), 우측 `자기 진단 데이터`(블루) + 가운데 화살표
- 가운데 화살표 위에 `Reframe` 또는 `뒤집기` 라벨 1개
- 하단 중앙에 Synapse 로고/워드마크만 배치 (시선 고정점)
- 텍스트는 브릿지 문장 1개 외 금지

### 브릿지 문장

> **"그 시청기록을 거꾸로 뒤집으면, 자기 진단 데이터가 된다."**

### 스크립트

> 우리는 이 질문에서 출발했습니다.  
> **시청 기록을 거꾸로 뒤집으면 어떻게 될까?**  
>
> 단순 로그가 아니라, **자기 진단 데이터**가 됩니다.  
> 무엇을 봤는지, 얼마나 깊이 봤는지, 어떤 채널·주제에 몰입했는지 —  
> 이걸 모으고 해석하면 **"지금의 나"**가 보입니다.  
>
> 그리고 거기서 한 걸음 더 나아가, **"되고 싶은 나"**, 즉 **이상향**을 설계하고,  
> **재생목록과 행동**으로 연결하는 시스템. 그게 **Synapse**입니다.
>
> 다음 슬라이드에서 이 흐름을 **한 장 루프**로 바로 보여드리겠습니다.

### 디렉션

- "거꾸로 뒤집으면"에서 **손 제스처** (뒤집기) 한 번
- 마지막에 **Synapse** 이름을 또렷하게
- 문제 키워드 재설명은 생략하고 **전환 선언**만 전달하기

## 슬라이드 5 · 한 장 요약 (0:45) — 발표자 A

### 슬라이드에 보여줄 것

핵심 루프 다이어그램 (가로 1줄):

```
시청기록 수집 → 21축 성향 진단 → 이상향 설계 → 재생목록·행동 실행
```

하단 한 줄:

> **웹앱 + 브라우저 확장 2클라이언트 · 백엔드 멀티에이전트 AI**

### 디자인 구상 (디테일)

- 핵심 루프는 **원형 루프**보다 **가로 4단 파이프라인**이 발표 속도에 유리
- 각 단계 아이콘 고정:
  - 수집(폴더/클라우드)
  - 진단(레이더 차트)
  - 이상향(타겟)
  - 실행(재생 버튼)
- 하단 서브 문장은 반투명 바 안에 넣어 보조 정보로 처리

### 스크립트

> Synapse가 하는 일을 **한 장**으로 요약하면 이렇습니다.  
>
> 첫째, **시청 기록을 수집**합니다. Google Takeout 업로드나 Drive 자동 연동,  
> 그리고 브라우저 확장으로 **웹 활동과 스크랩**까지 모읍니다.  
>
> 둘째, **21축 성향 진단**입니다. 심리학의 Schwartz 가치관 10축, TCI 기질 3축,  
> 그리고 행동 8축을 합쳐 **"지금의 나"**를 숫자와 페르소나로 보여줍니다.  
>
> 셋째, **이상향 설계** — AI와 대화하며 **되고 싶은 나**를 정의합니다.  
>
> 넷째, **재생목록과 행동 가이드**로 격차를 좁힙니다.  
> 분석에서 끝나지 않고, **실제 YouTube 재생목록 저장**까지 이어집니다.  
>
> 구조적으로는 **웹앱**과 **브라우저 확장** 두 클라이언트가 있고,  
> 백엔드는 **여러 AI 에이전트가 협업하는 멀티에이전트** 아키텍처입니다.

### 디렉션

- 다이어그램을 **왼쪽에서 오른쪽으로** 포인터로 따라가며 설명
- "21축"은 숫자만 말하고 세부 축 이름은 **6·8번에서** 풀기

## 슬라이드 6 · 시스템 아키텍처 ★ (1:30) — 발표자 A

> **승부처 1** — 면접관·청중이 기술 깊이를 판단하는 구간

### 슬라이드에 보여줄 것

에이전트 파이프라인 다이어그램 1장 (권장 레이아웃):

```
[Takeout / Drive] ──→ Indexer ──→ Profiler ──→ Navigator
                                      │              │
                                      ├─→ Curator ←──┘ (홈 채팅)
                                      │
[Extension] ──→ Archiver (맥락 Q&A)
     │
     └──→ Aggregator ──→ Reporter (B2B 트렌드)
```

강조 박스 2개:

- **LangGraph 상태 머신** — 단계별 분기·재시도
- **SSE 실시간 스트리밍** — 채팅·이상향 설계

면접관 질문 미끼 (작은 각주):

- *증분 diff · 배치 스코프 profiling*

### 디자인 구상 (디테일)

- 레이아웃: `입력 영역` / `에이전트 코어` / `출력 영역` 3개 스윔레인
- 메인 동선(Indexer→Profiler→Navigator)은 굵은 선, 보조 동선은 얇은 선
- `LangGraph 상태 머신`, `SSE 스트리밍`은 오른쪽 상단 배지로 고정
- 각주 영역에 작은 글씨로 `증분 diff`, `batch seal`, `retry loop` 표기
- 너무 복잡하면 2단계 애니메이션:
  1) 개인 루프 먼저
  2) Archiver/Aggregator/Reporter 추가

### 스크립트

> 이제 **어떻게 만들었는지**, 아키텍처를 보여드리겠습니다.  
> Synapse 백엔드의 심장은 **6개의 AI 에이전트 파이프라인**입니다.  
>
> **데이터가 들어오는 입구**는 **Indexer**입니다.  
> Google Takeout의 YouTube 시청 JSON을 파싱하고, **노이즈를 제거**하고,  
> 이전 분석과 **증분 diff**로 새로 본 영상만 골라냅니다.  
> YouTube Data API로 메타데이터를 보강하고, **임베딩**까지 넣어  
> `user_watch_catalog`라는 **시청 정본**을 만듭니다.  
> 여기는 의도적으로 **LLM 없이 규칙 기반**입니다. 빠르고, 비용이 적고, 재현 가능해야 하니까요.  
>
> Indexer가 끝나면 **Profiler**가 돌아갑니다.  
> 영상마다 **의미 분석** — 톤, 의도, 가치 신호 — 을 하고,  
> 그걸 모아 **21축 성향 프로필**과 **Portrait**, 즉 서술형 초상을 만듭니다.  
> 여러 파일을 한 번에 올리면 **배치(batch)** 로 묶고, 업로드가 끝날 때 **seal** 신호를 보냅니다.  
> 그러면 **그 배치에 속한 영상만** 스코프해서 Profiler가 **정확히 한 번** 돌아갑니다.  
> 중복 분석을 막고, "이번에 올린 데이터"만 반영하는 설계입니다.  
>
> 프로필이 나오면 갈래가 둘로 갈라집니다.  
> **Navigator**는 **이상향 3안**을 제안하고, 챗봇으로 대화하며 확정한 뒤,  
> **현재와 목표의 gap**을 비교하고, **맞춤 재생목록**까지 생성합니다.  
> **Curator**는 웹 홈의 **메인 AI 채팅**으로, DB 조회·벡터 검색·재생목록 생성 같은 **툴을 직접 호출**하는 오케스트레이터입니다.  
>
> 별도 축으로 **Archiver**가 있습니다. 브라우저 확장 사이드패널에서  
> **지금 보고 있는 페이지**를 읽고, 과거 스크랩과 **RAG**, 필요하면 **웹 검색**까지 붙여 Q&A합니다.  
> 근거가 부족하면 **스스로 재검색**하는 평가 루프가 있습니다.  
>
> 마지막으로 **Aggregator**는 플랫폼 전체의 스크랩·시청·행동 로그를  
> **6대 거시 도메인**으로 매핑해 **일별 트렌드**를 집계하고,  
> **Reporter**가 B2B용 **지식 그래프와 리포트**로 바꿉니다.  
>
> 이 모든 에이전트는 **LangGraph 상태 머신**으로 구현했습니다.  
> 단순 체인이 아니라, **분기·병렬·재시도**가 있는 워크플로우이기 때문입니다.  
> 사용자-facing 채팅은 **SSE 스트리밍**으로, 긴 분석 중에도 **진행 상태와 토큰**을 실시간으로 보냅니다.

### 디렉션

- **Indexer → Profiler → Navigator** 순으로 포인터 이동 (메인 스토리)
- "증분 diff", "배치 seal"은 **살짝 속도를 늦춰** — 질문 유도
- Archiver·Aggregator는 **짧게** — 8번에서 팀원이 풀어줌
- 1분 30초 넘기면 **Curator·Reporter 설명 축약**
